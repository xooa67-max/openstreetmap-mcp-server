# 0.1.2 — 2026-05-23

## Fixed

- **`NominatimService.buildCacheKey`**: SHA-256 hash replaces raw `JSON.stringify(params)` in `ctx.state` key — characters outside `[a-zA-Z0-9\-_./]` caused every Nominatim tool call to fail before reaching upstream APIs. ([#1](https://github.com/cyanheads/nominatim-mcp-server/issues/1))
- **`OverpassService.buildCacheKey`**: SHA-256 hash replaces raw Overpass QL string in `ctx.state` key — same invalid-character rejection, same fix.
