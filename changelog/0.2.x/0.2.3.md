# 0.2.3 — 2026-05-30

**openstreetmap_query_nearby**, **openstreetmap_query_bbox**, and **openstreetmap_query_raw** adopt the `ctx.enrich` API, surfacing agent-facing context in a typed `enrichment` block that reaches both `structuredContent` and the `content[]` markdown trailer.

## Changed

- **`openstreetmap_query_nearby` / `openstreetmap_query_bbox` enrichment** — `effectiveTag` (resolved `key=value` filter), `totalFound` (true upstream count before limit), `truncated` (boolean), and `notice` (empty-result guidance; absent when results returned) now surface in `enrichment`, reachable via both channels.
- **`openstreetmap_query_raw` enrichment** — `effectiveQuery` (the Overpass QL string as sent) and `notice` (empty-result guidance) surface in `enrichment`.
- **structuredContent field renames** (`query_nearby` / `query_bbox`) — `total_found` renamed to `totalFound`; `truncated` moved from `output` into `enrichment`. Values and presence are preserved in both channels; key names changed.
- **`@cyanheads/mcp-ts-core`** `^0.9.13 → ^0.9.16`.
